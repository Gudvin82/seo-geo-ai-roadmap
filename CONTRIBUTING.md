# Contributing

## Principles

- Prefer practical guidance over abstract commentary.
- Keep English and Russian structures aligned.
- Preserve the discoverability OS framing.
- Add validation, examples, or templates when adding new theory.

## Pull requests

1. Open or reference an issue.
2. Describe what changed and why it matters.
3. Note whether EN and RU were both updated.
4. Include validation notes for scripts or documentation links.

## Content quality bar

A contribution is stronger when it adds one or more of the following:

- an SOP that can be executed
- a checklist that can be used immediately
- a prompt that is copy-paste ready
- a template or example that reduces ambiguity
- a script that validates a repetitive task
