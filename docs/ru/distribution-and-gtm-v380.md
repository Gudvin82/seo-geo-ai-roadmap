# Distribution and GTM v3.8.0

`v3.8.0` делает репозиторий проще для внедрения, упаковки и продажи.

## Коммерческое обещание

Сильнее всего этот репозиторий работает как:

- self-hosted discoverability operating system
- workflow для аудита и повторного измерения
- движок клиентской выдачи и evidence
- база для AI-agent-friendly внедрения

## Use-case packs

- Agency: отдельный workspace на клиента, регулярная отчетность, graph
  walkthroughs, fix packs и двуязычные экспорты
- Founder: один self-hosted стек, один сайт, периодические аудиты и proof-first
  цикл улучшений
- In-house: общий truth center, provider-backed SoV checks и мониторинг
  регрессий
- Consultant: быстрый pre-sales audit, graph explanation и затем scoped backlog

## Что можно отдавать в продажу и delivery

- executive summary
- fix pack
- graph JSON snapshot
- client delivery pack
- weekly или monthly operations update

## Каноническая последовательность

1. исследование через `/geo quick` или `/geo audit`
2. объяснение через `/geo graph`
3. приоритизация через `/geo report`
4. улучшение через content, schema, technical и trust work
5. повторное измерение через `/geo compare`

## Граница open-source и commercial

Open-source core:

- методология
- self-hosted продукт
- скрипты
- документация
- шаблоны
- graph UX

Commercial или service layer:

- ваш хостинг
- ваши операторы
- ваш client service
- ваши кастомные интеграции
- ваша delivery-модель

Репозиторий дает движок. Коммерческий результат все равно требует исполнения.
