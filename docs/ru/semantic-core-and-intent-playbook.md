# Semantic Core and Intent Playbook

Этот playbook нужен, потому что хорошая discoverability — это не только
technical layer.

## Что он покрывает

- search intent clustering
- page-to-query mapping
- service и location modifiers
- support layer из expert, trust и proof content
- answer-ready content design для AI surfaces

## Минимальный workflow

1. перечислить target services, products, problems и geographies
2. сгруппировать запросы по intent, а не только по словам
3. назначить одну primary page на каждый intent cluster
4. найти content gaps и page collisions
5. добавить proof и FAQ support там, где высоки trust-risk и ambiguity

## Типовые ошибки

- одна страница пытается ранжироваться сразу по нескольким несовместимым intents
- city, service и use-case modifiers перемешаны случайно
- нет явной proof page под коммерческие claims
- контент есть, но ни одна страница не владеет intent полностью

## Как выглядит сильная semantic structure

- одна ясная promise на страницу
- один primary intent cluster на страницу
- related secondary intents поддерживаются, не размывая core promise
- есть definitions, examples и proof, если тема сложная
- internal links отражают semantic hierarchy

## GEO и AI последствия

Semantic clarity помогает AI, потому что:

- ассистенты увереннее суммируют чистые сущности
- answer surfaces лучше воспринимают страницы с явной promise и supporting proof
- FAQ и definition blocks снижают factual drift и ambiguity
