# Сводка релиза v6.7.0

`v6.7.0` сфокусирован на productization документации и большей ясности
onboarding path.

## Что изменилось

- пересобран root experience для английского и русского README
- добавлены visual roadmap и i18n-status tracker
- добавлены явные guidance по support и security
- добавлен английский alias entrypoint через `README_EN.md`
- переписан проектный roadmap вокруг утвержденного пути `v6.7.0` → `v6.8.5`

## Почему это важно

До `v6.7.0` у репозитория уже был сильный методологический и продуктовый
foundation, но root docs все еще ощущались скорее как мощный repo, чем как
чистая learning и execution system.

После `v6.7.0` новым пользователям и AI coding agents проще:

- быстрее находить правильный entrypoint
- быстрее понимать текущий roadmap и language parity
- видеть более явные support, security и contribution paths
- воспринимать root docs как полноценный onboarding layer, а не как частичную карту

## Честная граница

`v6.7.0` усиливает зрелость документации, а не hosted SaaS maturity.
