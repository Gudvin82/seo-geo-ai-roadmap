# v6.0.0 Release Summary

`v6.0.0` is a trust-density release.

It does not try to win by adding even more surface area. It tries to reduce the
gap between packaging and substance.

## What changed

- new root-level methodology explanation
- transparent scoring explanation linked to real code paths
- new AI task packs for audit, deployment, improvement, productization, and public evaluation
- deeper practical playbooks for:
  - technical SEO
  - semantic core and intent
  - competitor and authority gaps
  - GEO and AI operations
- upgraded core checklists so they are explicitly quick execution cards instead of pretending to be the full methodology
- cleaner docs routing through current entrypoints plus explicit archive policy
- new bounded public audit case for `anmalishev.ru`

## Why this release matters

Before `v6.0.0`, the repo was strong but easier to misread as:

- overpackaged
- too release-note-heavy
- more impressive in breadth than in density

`v6.0.0` responds directly to that perception by making:

- the core path clearer
- the scoring more explainable
- the playbooks more practical
- the AI handoff more concrete
- the evidence layer more visible

## Boundary unchanged

`v6.0.0` still does **not** claim:

- hosted SaaS run by the maintainer
- perfect or universal scoring
- finished enterprise SEO methodology across every subdiscipline
- autonomous production edits without review
