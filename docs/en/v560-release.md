# v5.6.0 Release Summary

`v5.6.0` upgrades the project from a strong self-hosted SaaS-ready foundation
into a more production-shaped operator system for integrations, RU market
execution, and social-commerce intelligence.

## What changed

- Added integration runtime profiling so every integration can now expose:
  refresh cadence, retry policy, token-rotation expectations, runtime level,
  and recovery guidance.
- Added a dedicated integration runtime center in the API and frontend.
- Added deeper RU market surfaces:
  `yandex_neuro`, `vk_organic`, `telegram_channels`, `dzen`, and `rutube`.
- Added a dedicated RU market command center that combines:
  Yandex search, Yandex local, RU AI readiness, RU social distribution, and
  local trust overlays.
- Expanded social and commercial operator loops so community demand can be
  turned into:
  FAQ blocks, trust blocks, proof strips, objection handling, and sales-ready
  messaging assets.
- Clarified that the project remains free and self-hosted by default, so
  billing is not required to call it SaaS-ready for operator-owned deployments.

## Why it matters

This release makes the repo more useful for agencies, founders, and in-house
operators who need more than a methodology:

- they can see which integrations are still starter-level,
- which ones are closer to managed runtime behavior,
- how RU demand and local trust should be interpreted,
- and how social signals should feed real deliverables and commercial content.

## Honest boundary

`v5.6.0` still does **not** mean:

- a maintainer-hosted public SaaS,
- mandatory billing or subscriptions,
- enterprise SSO or SCIM out of the box,
- or fully autonomous no-review publishing.

It does mean a stronger self-hosted and operator-controlled product layer that
is much closer to real recurring delivery.
