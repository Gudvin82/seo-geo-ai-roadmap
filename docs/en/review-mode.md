# Review Mode

Review mode explains what the platform may automate, what requires approval,
and what must not be auto-applied.

## Can be automated

- inventory sync
- starter data import
- artifact generation
- draft patch packs
- exportable client-pack assembly

## Requires human review

- title rewrites
- schema changes
- public-facing claims
- white-label framing
- human-approved publish
- client delivery wording

## Must not be auto-applied

- silent destructive changes
- publishing without review
- legal or safety-sensitive claims without approval

## Intended operator stance

This mode is designed to help both humans and AI agents understand the approval
boundary. `v3.1.0` expands operational output, but it does not remove the need
for review before publication, client delivery, or sensitive factual claims.
