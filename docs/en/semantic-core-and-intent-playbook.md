# Semantic Core and Intent Playbook

This playbook exists because good discoverability is not only technical.

## What this playbook covers

- search intent clustering
- page-to-query mapping
- service and location modifiers
- expert, trust, and proof content support
- answer-ready content design for AI surfaces

## Minimum workflow

1. list target services, products, problems, and geographies
2. group queries by intent, not only by wording
3. assign one primary page per intent cluster
4. identify content gaps and page collisions
5. add proof and FAQ support where trust or ambiguity is high

## Typical mistakes

- one page tries to rank for several incompatible intents
- city, service, and use-case modifiers are mixed randomly
- there is no clear proof page for commercial claims
- content exists, but no page owns the search intent

## What strong semantic structure looks like

- one clear promise per page
- one primary intent cluster per page
- related secondary intents supported without muddying the core promise
- explicit definitions, examples, and proof when the topic is complex
- internal links reflect the semantic hierarchy

## GEO and AI implications

Semantic clarity helps AI because:

- assistants summarize cleaner entities more confidently
- answer surfaces prefer pages with explicit promise and supporting proof
- FAQ and definition blocks reduce factual drift and ambiguity

## Outputs

- intent map
- page ownership map
- content-gap backlog
- FAQ and proof backlog
- hub and internal-linking plan
