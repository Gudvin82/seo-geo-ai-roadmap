# v6.8.0 Release Summary

`v6.8.0` strengthens the repository's proof, case, and working-tool layer.

## What changed

- added `scripts/case_library_builder.py`
- added `scripts/synthetic_case_builder.py`
- added `scripts/issue_pack_generator.py`
- expanded case and proof docs with synthetic-case and issue-pack workflows
- expanded proof kit and evidence lab surfaces with stronger case-library and
  export readiness signals
- added stronger issue templates and richer proof/case templates

## Why it matters

Before `v6.8.0`, the repository already had bounded public cases and strong
proof primitives, but the repeatable tooling around them was still thinner than
the methodology itself.

After `v6.8.0`, teams can more easily:

- train on synthetic but clearly labeled cases
- build a case library index
- convert findings into issue packs for implementation teams
- keep proof, cases, and exports more structured and repeatable
