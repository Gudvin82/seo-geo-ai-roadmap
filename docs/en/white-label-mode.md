# White-Label Mode

Use workspace branding fields when you need a client-facing layer on top of the
same shared platform.

## Current white-label fields

- configurable app or client-facing title
- `client_report_title`
- `client_report_subtitle`
- `client_report_footer`
- `branding_logo_url`

## Recommended workflow

1. Create one workspace per client.
2. Fill white-label fields before exporting reports.
3. Keep one brand-facts profile per site or per business line.
4. Generate client packs only after at least one reviewed audit exists.
5. Export artifacts only after human review.

## What this release is and is not

`v3.1.0` ships a stronger white-label starter, not a fully isolated reseller
control plane. It is intended to help agencies and operators produce cleaner
client deliverables without hiding the transparent, self-hosted foundation.
