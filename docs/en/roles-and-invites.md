# Roles and Invites

## Workspace roles

- `owner`: full control, workspace governance, role changes
- `admin`: can invite users, manage providers, inspect logs
- `editor`: can create projects, facts, and audit runs
- `viewer`: can inspect projects, reports, and artifacts

## Invite flow

1. Admin or owner creates an invite.
1. The platform returns an invite token.
1. The invited user accepts it while signed into the matching email.
1. The membership is created and audit logs are updated.
