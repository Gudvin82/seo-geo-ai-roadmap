# v6.7.0 Release Summary

`v6.7.0` focuses on documentation productization and onboarding clarity.

## What changed

- rebuilt the root English and Russian README experience
- added a visual roadmap and i18n-status tracker
- added explicit support and security guidance
- added an English alias entrypoint through `README_EN.md`
- rewrote the project roadmap around the approved `v6.7.0` → `v6.8.5` path

## Why it matters

Before `v6.7.0`, the repository already had a strong methodological and product
foundation, but the root docs still felt more like a powerful repo than a clean
learning and execution system.

After `v6.7.0`, new users and AI coding agents can:

- find the right entrypoint faster
- understand the current roadmap and language parity faster
- see clearer support, security, and contribution paths
- treat the root docs as a proper onboarding layer instead of a partial map

## Honest boundary

`v6.7.0` improves documentation maturity, not hosted SaaS maturity.
