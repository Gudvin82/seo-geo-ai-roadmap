# How It Works

`v3.4.0` makes the repository easier to operate by exposing a clearer modular
flow.

## Flow

1. Bootstrap the stack or select a running environment.
2. Choose the command surface: `audit`, `llmstxt`, `report`, `compare`, and so on.
3. Route into scripts, docs, and API entrypoints with the command catalog.
4. Run checks, collect evidence, and generate reports or delivery packs.
5. Schedule repeated runs when the work becomes ongoing.

## Main modules

- bootstrap and deployment
- routing and command surface
- scripts and validators
- backend API and audit logs
- frontend operator console
- docs, templates, and delivery artifacts

## Why this matters

The repository is easier to evaluate and adopt when the first step is obvious.
