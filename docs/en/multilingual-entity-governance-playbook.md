# Multilingual Entity Governance Playbook

## Goal

Keep one brand entity coherent across multiple languages and markets.

## Inputs

- canonical global fact sheet
- local market adaptations
- hreflang map
- schema and profile references by language

## Step sequence

1. Define canonical global facts.
2. Define allowed local adaptations.
3. Compare homepage, market pages, and local profile pages.
4. Align structured data, hreflang, and fact wording.
5. Review market-specific AI and search surfaces separately.

## Expected outputs

- one global truth source
- explicit local deviation log
- aligned schema and hreflang references
- per-market verification notes

## What to measure

- fewer contradictions between EN and RU pages
- cleaner entity identity in profile and AI surfaces
- reduced localization drift over time

## Risks

- literal translation where local commercial language should differ
- local teams changing facts without updating the canonical source
- using different company descriptors in different markets

## Minimum viable path

If governance time is limited, align first:

1. legal entity name
2. offer wording
3. geography and service scope
4. founder or expert identity
5. primary trust pages
