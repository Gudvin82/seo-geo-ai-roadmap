# Site Adaptation Playbooks

## Local business site

- create a workspace
- create a project with local market and language
- run local/Yandex readiness, facts, and llms checks
- draft implementation tasks from findings

## Expert / personal brand site

- emphasize truth center and canonical facts
- run hallucination and entity hierarchy checks
- export report for ongoing content governance

## Product / service site

- run technical, `llms.txt`, freshness, and SoV starter checks
- map findings into page briefs and implementation backlog

## Multilingual site

- set language and market explicitly
- reuse prompts and checklists in EN and RU
- compare entity and factual consistency across language layers
