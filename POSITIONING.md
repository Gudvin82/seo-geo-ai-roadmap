# Positioning

## One-line statement

`seo-geo-ai-roadmap` is a free, transparent, self-hosted discoverability
platform and methodology for teams that need one operating system across
technical SEO, GEO, AI visibility, factual consistency, and bilingual reporting.

## Three outcome layers

The repo is intentionally built around three connected outcomes:

- Rankings: classical SEO visibility
- AI citations / AI visibility: GEO/AEO surface presence and citation quality
- Conversion and trust: leads, pipeline, branded demand, and human confidence

The project is strongest when users treat these as linked but non-identical
layers.

## Primary ICPs

- Agency operators managing multiple client sites
- In-house teams that need repeatable audits and evidence-backed prioritization
- Founders and expert consultants running their own sites and products

## What makes it different

- It is self-hosted first, not cloud-first with an open-source wrapper
- It treats English and Russian as production layers, not translation leftovers
- It combines methodology, platform, and operator workflow in one repo
- It is built to be usable by humans and by AI coding agents
- It measures and re-runs, instead of only describing best practices
- It treats GEO/AI as measurable, scenario-based, and anti-hype rather than as
  a vague AI add-on

## Framework vs platform vs service system

- Framework: docs, prompts, templates, checklists, and scripts
- Platform: the app, APIs, auth, workspaces, providers, audits, reports, SoV
- Service system: how teams actually deliver audits, prioritize fixes, and
  compare deltas over time

## What users should understand immediately

### In 15 minutes

You can deploy the stack, create a workspace, run an audit, and export a report.

### In 30 days

You can establish a repeatable audit and reporting routine with visible deltas.

### In 90 days

You can run agency, in-house, or founder workflows from the same operator base.

## What this is not

- not a replacement for strong technical SEO
- not a black-box rank tracker
- not a hosted SaaS dependency
- not a promise of guaranteed AI mentions
- not a generic "AI growth" wrapper with hidden scoring

## Product promise

If a user gives this repo to Codex, Claude Code, Cursor, or another AI coding
agent, the agent should be able to:

1. deploy the project
2. connect providers
3. run a real audit
4. produce reports and artifacts
5. prioritize fixes
6. re-run and compare deltas

## Trust stance

This repository prefers bounded proof over exaggerated claims:

- transparent metrics
- explicit limitations
- visible screenshots and outputs
- verifiable commands
- repeatable EN/RU documentation
