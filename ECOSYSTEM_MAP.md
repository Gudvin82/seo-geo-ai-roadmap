# Ecosystem Map

This repository does not replace adjacent products. It explains how to use them inside a discoverability stack.

## OpenSEO

- What it is: an open-source SEO platform concept for audits, tracking, and workflow support.
- Strong side: centralizing SEO operations and diagnostics.
- RU/CIS use: useful where teams want more control than SaaS tools provide.
- Why we do not replace it: this repository explains process, governance, AI discoverability, and bilingual operating logic around such tooling.

## Perplexica

- What it is: a self-hosted AI answering engine.
- Strong side: controllable AI-answering workflows and experimentation.
- RU/CIS use: useful for internal testing where teams want self-hosted experimentation.
- Why we do not replace it: we describe how to test discoverability and citations, not how to run an answering engine itself.

## Open WebUI

- What it is: a self-hosted AI platform and user interface.
- Strong side: team access to local or hosted models with shared workflows.
- RU/CIS use: practical for internal AI operations and document-based testing.
- Why we do not replace it: this repository provides playbooks and prompts for discoverability work that can be executed inside Open WebUI.

## Trieve

- What it is: a search, RAG, recommendations, and analytics API platform.
- Strong side: developer-friendly retrieval and search infrastructure.
- RU/CIS use: relevant where teams build searchable knowledge hubs or answer systems.
- Why we do not replace it: our repository explains how content structure, entities, and monitoring should feed systems like Trieve.

## Onyx

- What it is: an enterprise AI search and chat platform.
- Strong side: secure internal knowledge retrieval and enterprise-grade access control.
- RU/CIS use: relevant for private knowledge bases or internal enablement layers.
- Why we do not replace it: we cover external discoverability and execution governance, not internal enterprise search product features.

## Flowise

- What it is: a visual builder for LLM and agent workflows.
- Strong side: quick orchestration of prompt pipelines, retrieval, and automation.
- RU/CIS use: useful for agencies and internal teams that want rapid prototyping.
- Why we do not replace it: this repository defines the operating manual and validation logic that can be implemented using Flowise.
