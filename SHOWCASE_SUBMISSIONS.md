# Showcase Submissions

Use this file when you want to understand what a strong public submission looks
like for the repository.

## Good submission types

- bounded public case
- synthetic training case
- launch material improvement
- deployment friction report
- script-first operator workflow improvement

## Good submission checklist

- clear fact vs inference boundary
- reusable operator lesson
- explicit market context
- links to proof assets or docs
- safe public wording
