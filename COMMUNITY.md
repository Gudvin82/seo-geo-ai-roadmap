# Community

This file is the shortest public participation surface for the repository.

Use it when you want to:

- suggest a practical improvement
- submit a bounded public case
- propose a synthetic training case
- share a launch, deployment, or audit story

## Best ways to participate

1. Open a GitHub issue.
2. Use the matching template:
   - content suggestion
   - proof or case request
   - bug report
3. State whether your input is:
   - public fact
   - bounded rollout record
   - synthetic training material
   - internal-only and not publishable

## What helps most

- before/after evidence with clear boundaries
- reproducible script or deployment findings
- EN/RU parity improvements
- launch feedback from real operators and AI agents
- better onboarding or contribution clarity
