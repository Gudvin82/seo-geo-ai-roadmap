# Методология v6

Это самый короткий и честный ответ на вопрос, что именно представляет собой
методология репозитория.

Это не:

- волшебная формула роста
- скрытая scoring-модель
- black-box crawler, который заменяет мышление оператора
- исчерпывающая SEO-энциклопедия в одном файле

Это:

- практический operating framework для SEO, GEO и AI discoverability
- self-hosted система из app, scripts и docs
- способ превращать аудит в доказательства, приоритеты, исправления и повторные проверки
- двуязычный workflow для глобального и русскоязычного рынка

## Операционная модель

Репозиторий смотрит на discoverability как на три связанных слоя:

1. Rankings
   Видимость в поиске, crawlability, indexation, техническая гигиена,
   семантика, авторитетность и local search execution.
2. AI visibility
   Может ли ассистент или answer engine правильно распознать сущность,
   извлечь факты и процитировать сайт или бренд.
3. Conversion and trust
   Попадает ли трафик или citation на страницы, которые понятны, вызывают
   доверие, коммерчески пригодны и не противоречат реальности.

Максимальную ценность репозиторий дает, когда эти три слоя управляются вместе.

## Что входит в методологию

### 1. Audit system

- technical SEO
- информационная архитектура
- семантика и intent
- контент, E-E-A-T и entity clarity
- local SEO и Яндекс-специфика
- GEO и answer-surface readiness
- factual consistency и drift control
- отчетность, proof и export задач

### 2. Execution system

- генерация findings и score
- приоритизация через impact / effort / confidence
- task bundles для owner-ready delivery
- объяснение зависимостей через graph
- approval-first CMS и delivery flow
- repeat audit и compare loop

### 3. AI-agent system

- task packs
- prompt packs
- command surface
- machine-readable contracts
- self-check path перед заявлением “готово”

## Что пока покрыто не до предельной глубины

В репозитории эти зоны уже есть, но пока не на уровне “финальной агентской библии”:

- enterprise-grade competitive intelligence в большом масштабе
- log-file SEO как first-class runtime
- зрелая система link acquisition
- полностью автоматизированный AI brand monitoring по всем провайдерам
- hosted-SaaS удобство без operator setup

Поэтому репозиторий надо читать как:

- сильный framework и operating platform
- а не как заявление, что каждая поддисциплина SEO уже исчерпывающе закрыта

## Как правильно читать чеклисты

Папка `checklists/` специально сделана короткой.

Эти файлы — это:

- execution cards
- QA-напоминания
- краткие handoff summary

Сами по себе они не являются всей методологией.

Для серьезной работы нужно сочетать:

- один deep playbook
- один checklist
- один prompt или task pack
- один script или app flow
- один proof или case reference
