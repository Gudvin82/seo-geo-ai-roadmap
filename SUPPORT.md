# Support

## What To Use This File For

Use this file when you want to know where to ask for help, what kind of help is
reasonable, and how to report something clearly.

## Good Support Requests

- a broken documented path
- an unclear onboarding step
- a failed script, test, or demo path
- a mismatch between docs and implementation
- a question about which entrypoint to use

## Best Current Support Path

1. Read the relevant entrypoint first:
   - [DOCS_INDEX.md](./DOCS_INDEX.md)
   - [WALKTHROUGH.md](./WALKTHROUGH.md)
   - [START_HERE_FOR_AI.md](./START_HERE_FOR_AI.md)
2. If the problem is concrete, open a GitHub issue.
3. If the problem is security-sensitive, use [SECURITY.md](./SECURITY.md).

## What To Include In A Good Issue

- what you tried
- which file or path you followed
- what you expected
- what actually happened
- relevant logs, screenshots, or command output

## Current Boundaries

This repository is open-source and self-hosted first.

It does not currently promise:

- SLA-backed support
- guaranteed turnaround time
- custom consulting through the repository itself

## Language

Russian and English reports are both welcome.
